Install¶

Dependencies¶

  • clone the repository

  • enter the repository folder

  • now choose one of the following cases depending on your precondition:

No pre-exisiting conda environment¶

  • install dependencies using a conda environment with conda env create --file environment.yml

Note

Using conda env create, you create an virtual environment with it’s isolated python interpreter and packages. The dependencies for this repository are listed in the environment.yml.

  • activate the new environment mdenv using conda activate mdenv

Note

This is nesessary every time you start the terminal to run the app. The command line should start with (mdenv) if the needed environment is active.

  • finally run the app with python khapp.py

Update existing conda environment¶

  • activate the environment you want to update: conda activate <your_environment>

  • install dependencies using: conda env update --file environment.yml

Note

Updating the dependencies using the environment.yml can lead to wrong package versions for your code.

Installation¶

  • open a console in <path>/measurement-data/data_tools and enter python deploy.py [--version VERSION]

Note

You can pass a desired version of datatools using the --version flag. The available versions can be shown with the git tag command. If version is left out the master will be checked out.
