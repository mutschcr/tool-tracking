Load data with the MeasurementDataReader¶

Set up connection to the data source¶

First we have to connect to our pre-defined/registered data source using the main class:

>>> from datatools import MeasurementDataReader, Measurement, MeasurementSeries
>>> mdr = MeasurementDataReader(source='my-data-source')

The source can be the name a registered data source (recommended) or the path to your local data directory, e.g.: C:/Users/wllr/lze-kh-data. If it is not set the registered default source will be used!

Create a Query to load Measurements or Actions¶

Using the mdr object we can create a Query object to formulate a query to the data source.

>>> q = mdr.query(query_type=Measurement)

Depending on the use case you can query for Measurement s which represents a temporal contiguous time series with inhomogeneous labels or Action s which are parts of Measurement s but have the same label throughout their time range.

>>> from datatools import Action
>>> q = mdr.query(query_type=Action)

Narrow a Query to load data of interest¶

>>> q.filter_by(MeasurementSeries == "pythagoras-04-20200219")

The number of measurements that this query applies to can be determined before actual data is read.

>>> q.count()
2

Note

For a comprehensive overview on how to make queries see the chapter Query Interface in Detail.

Get measurement data¶

Finally the data can be obtained with:

>>> data_dict = q.get()  # mdr.read() is deprecated

[INFO] Preparing data from:
  ..\measurement-data_new\electric_screwdriver_2_stage\pythagoras-04-20200219
  ..\measurement-data_new\electric_screwdriver_2_stage\pythagoras-04-20200219
[INFO] Read annotation: 100%|██████████| 6/6 [00:00<00:00, 46.13it/s, file=data-02.annotation]
[INFO] Finished with 2 measurement(s).

The selected measurement series (pythagoras-04-20200219) contains 2 datasets with data from the accelerometer, gyroscope and the magnetic field sensor. Each dataset is identified by a double-digit ID.

  • The data_dict returned by the Query is a dict containing the two datasets as DataBunch which can be accessed by their IDs as keys.

  • The DataBunch class stores all the data for one single measurement or action. To see which data is available check with info:

>>> data_bunch = data_dict['01']
>>> data_bunch.info
['acc', 'gyr', 'mag', 'classes']

Here we have four keys:

  • acc/gyr/mag: contain acc, gyr, mag data and the corresponding label/target vector (encoded with values between 0 and n_classes-1)

  • classes: Dict[int, str] -> a dictionary of classes used for labelling

If audio data is available there is an additional key:

  • mic: contain audio data and the corresponding label/target vector (encoded with values between 0 and n_classes-1)

>>> acc_data = data_bunch.acc
>>> type(acc_data)
datatools.types.DataCSV

The actual measurement data is stored using DataCSV which is actually a pandas.DataFrame with some useful extensions, but can be handled like an ordinary DataFrame.

>>> data_bunch.acc

Note

If there is only one dataset, a single DataBunch is returned instead of a dict!
