Query Interface in Detail¶

As we saw before one can create a query object using the MeasurementDataReader:

from datatools import MeasurementDataReader, Measurement
mdr = MeasurementDataReader(source='my-data-source')
q = mdr.query(query_type=Measurement)

Also have a look on the api documentation of the Query class. Now we want to have a closer look on how to formulate a precise query.

Build Keyword Expressions¶

The main tool to formulate a query are keyword expressions. They always have the following syntax:

<KEYWORD(.SPECIFIER)> <comparison operator> <VALUE>

Examples:

MeasurementSeries == "pythagoras-04-20200219"
MeasurementSeries.date > "2020-02-19"
Measurement.work_piece == ["engine block", "suitcase"]

Valid VALUES to compare with can be strings and also lists of strings depending on the use case. There are multiple filter objects to formulate such keyword expressions:

from datatools import Tool, Config, MeasurementSeries, Measurement, DataTypes, Action

From the example above we can see that filter objects can have specifiers to state the query more precisely. For information about the special filters DataTypes and Action see next section. If no specifier is stated the default one is used. The available specifiers and default values for each object are:

filter object

specifiers

default

Tool

[‘name’, ‘id’, ‘model’, ‘model_id’]

name

Config

[‘rpm_cw’, ‘rpm_ccw’, ‘torque’, ‘air_pressure’]

MeasurementSeries

[‘name’, ‘date’, ‘module’, ‘campaign_id’]

name

Measurement

[‘id’, ‘person’, ‘work_piece’]

id

Attention

  • Invalid use of comparison operators may lead to errors or wrong results.

  • Do not mistake the equal operator (==) for the assignment operator (=)

Special Filter Objects¶

To filter data based on the sensor data type one not only needs the special filter object DataTypes but also data type objects.

from datatools import ACC, GYR, MAG, MIC, POS, VEL

With that the keyword expressions can be formulated as before:

DataTypes == [ACC, MIC]
DataTypes == GYR

To build valid keyword expressions with the Action object it is necessary to change the query_type when creating a query object otherwise this expression has no effect:

q = mdr.query(query_type=Action)

Similar to the DataTypes object the Action object has no specifiers:

Action == "tightening"
Action == ["motor_activity_ccw", "motor_activity_cw"]

Filter using Keyword Expressions¶

The last step before we can get the desired data is to apply the formulated keyword expressions to the query object using its filter_by function:

q.filter_by(
    DataTypes == ACC,
    Measurement.id == ["01", "02"],
    Measurement.work_piece == ["engine block", "suitcase"]
)

Note

The keyword expressions will linked by a logical conjunction (and) by default. If you want to change this behaviour and use a logical disjunction (or), you have to pass an additional keyword argument join="or" to the filter_by function.

To see the query result before reading the actual data the df attribute of the query object can be used to get a pandas DataFrame with all matching Measurements.
